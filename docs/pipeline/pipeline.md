# Pipeline

## 목표
크롤링으로 수집한 원천 데이터를 RAG 검색과 리포트 생성에 사용할 수 있는 사례 데이터셋으로 정리한다.

## 기본 흐름
1. 소스별 크롤링 / 수집
2. 원문 저장
3. 정제 및 필드 표준화
4. 엔티티 추출
5. 중복 제거
6. 문서 chunk 분할
7. 임베딩 생성
8. PostgreSQL + pgvector 적재
9. 검색 / 시각화용 부가 메타데이터 생성

## 정제 단계에서 처리할 것
- HTML / 잡음 제거
- 가격 / 날짜 / 플랫폼 정보 정규화
- 계좌 / 전화 / 메신저 ID 추출
- 제목 / 본문 / 태그 구조화

## 적재 대상
- `cases`
- `case_chunks`
- `case_entities`
- `seller_observations`

## 운영 전략
- 초기 단계: 수동 실행 가능한 Python 스크립트 중심
- 후속 단계: Airflow 배치화 검토
- 대규모 이벤트 처리 필요 시 Kafka 검토

## 메모
- 장기적으로는 비식별화와 품질 검증 단계를 더 강화한다.
- 시각화 품질을 위해 사례 라벨과 이웃 관계 정보를 함께 준비한다.
