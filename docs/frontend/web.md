# Web

## 역할
웹 리포트 페이지는 extension에서 클릭해서 이동하는 **report page** 이며, 확장 프로그램이 제공하는 짧은 경고를 **상세 분석 결과**로 확장해서 보여주는 화면이다.

## 주요 화면

### `/`
- 프로젝트 소개
- 간단한 사용 방법
- 리포트 진입 안내
- extension 과의 연결 방식 설명

### `/report/:scanId`
- 위험 수준 / 위험 점수
- 근거 문장 목록
- 유사 사례 요약
- 대응 가이드
- 판매자 관찰 이력 요약
- extension 의 `scan panel` 에서 보던 요약을 더 자세히 풀어주는 영역
- 임베딩 공간 시각화 요약 카드
- 사용자 맥락 기반 주의도 조정 카드
- 전체 사용자 통계 대비 비교 카드

### `/report/:scanId/visual`
- UMAP 2차원 시각화
- 현재 게시글과 유사 사례의 거리 / 위치 비교
- k-NN 기반 이웃 사례 보기
- 간단한 통계 패널
- 사기 게시글 군집과 현재 게시글의 상대적 위치 설명
- “가깝다 / 멀다”를 일반 사용자가 이해할 수 있는 문장형 안내

### `/help`
- 사기 대응 가이드
- 신고 안내

### `/login` *(후순위)*
- 향후 보호 프로필 기능을 위한 로그인 진입점

### `/profile` *(후순위)*
- 향후 개인 보호 프로필 관리 화면

## 상태 관리 방식
- **React Router**: 라우팅
- **TanStack Query**: 서버 상태, polling, 캐시
- **Zustand**: UI 로컬 상태 (선택한 점, 필터, 패널 열림 상태 등)
- **fetch wrapper**: API 호출은 우선 `fetch` 기반으로 단순하게 유지

## API 호출 방식
- `POST /api/v1/scans` 로 scan 작업 생성
- `GET /api/v1/scans/{scan_id}` 로 상태 / 결과 조회
- `POST /api/v1/scans/{scan_id}/feedback` 로 피드백 전송

## 시각화 메모
- 기본 시각화는 **UMAP 2D** 기준으로 잡는다.
- t-SNE는 비교 실험 후보로 둔다.
- 시각화에는 다음 정보를 함께 표시할 수 있다.
  - 현재 게시글 위치
  - 유사 사례 위치
  - 라벨별 색상 구분
  - nearest neighbor 강조
  - 간단한 통계 카드
- 대시보드는 “시각화가 예쁘다”보다 “사용자가 위험도를 직관적으로 이해한다”를 우선한다.
- 따라서 시각화에는 아래 설명 요소를 함께 둔다.
  - 현재 게시글이 사기 사례 군집과 얼마나 가까운지에 대한 한 줄 설명
  - 현재 게시글과 정상/의심 사례 평균 거리 비교
  - nearest neighbor 몇 개가 어떤 이유로 가까운지에 대한 보조 설명
  - 축 자체보다 상대적 거리 해석에 집중하는 범례와 도움말

## 대시보드 확장 메모
- `report page` 는 단순 결과 나열이 아니라 **대시보드** 역할을 하도록 확장한다.
- 우선 후보 영역:
  - 위험 요약 카드
  - 임베딩 공간 시각화 카드
  - 판매자 / 작성자 관찰 정보 카드
  - 사용자 맥락 기반 주의 카드
  - 전체 사용자 통계 비교 카드
  - 외부 조회 / 신고 연계 카드
- 특히 임베딩 공간 시각화는 “사기 게시글과 얼마나 가깝고 먼지”를 일반 사용자가 이해할 수 있도록 문장, 범례, 예시를 같이 제공하는 방향으로 설계한다.

## 개인 보호 프로필 메모
향후 로그인 기능을 붙인 뒤, 사용자가 선택적으로 보호 프로필을 설정할 수 있게 설계한다.
예시:
- 연령대
- 과거 피해 경험 여부
- 경고 민감도
- 중고 거래 경험 수준
- 반복 입금 / 추가 송금 요구에 대한 취약 여부

> 현재 단계에서는 로그인 / 인증을 구현하지 않는다.

## 사용자 맥락 활용 메모
- 로그인 기능이 붙으면 대시보드에서 사용자 맥락을 활용할 수 있다.
- 예시:
  - 중고 거래 경험이 적은 사용자에게 더 보수적인 경고 기준 적용
  - 과거 피해 경험이 있거나 경고 민감도를 높게 설정한 사용자에게 더 강한 주의 문구 제공
  - “두 번 입금해 달라” 같은 요구를 그대로 따르기 쉬운 사용자를 위한 추가 설명 카드 제공
- 이 정보는 탐지 결과를 바꾸는 것보다 **설명 강도와 안내 방식**을 조정하는 데 우선 사용한다.

## 작성자 / 통계 활용 메모
- 작성자 최근 거래 내역, 판매 패턴, 반복 관찰 이력은 `report page` 대시보드에서 별도 카드로 보여줄 수 있다.
- DB 에 저장된 전체 통계와 비교하는 화면도 후보로 둔다.
- 예시:
  - 동일 판매자 / 계좌 / 연락처 관찰 횟수
  - 최근 거래 빈도와 일반 판매자 평균 비교
  - 현재 게시글 위험 점수가 전체 데이터 분포에서 어느 구간인지 표시

## 외부 조회 / 신고 연계 메모
- 경찰청 또는 더치트 연계는 `report page` 에서 별도 카드로 두는 방향을 고려한다.
- 초기에는 자동 신고보다 아래 수준을 우선한다.
  - 공식 조회 / 신고 페이지로 이동
  - 필요한 입력값을 미리 정리해주는 가이드
  - 조회 결과를 사용자가 직접 확인하도록 돕는 보조 흐름
- 이후 후보:
  - Playwright 기반 자동 조회 보조
  - 더치트 / 경찰청 조회 결과를 `report page` 에 가져와 비교 표시
- 다만 외부 서비스 약관, 인증, 자동화 가능 여부를 먼저 검토해야 한다.

## 디자인 메모
- 와이어프레임과 시안은 **Figma** 에서 먼저 설계한다.
- 실제 구현은 React + Tailwind CSS 기준으로 옮긴다.
- 역할 분리 원칙:
  - 짧은 경고와 즉시 하이라이트는 extension `scan panel`
  - 추가 질의응답은 extension `chat panel`
  - 긴 설명과 상세 분석은 web `report page`
- 대시보드 카드는 “전문가용 분석 화면”보다 “비숙련 사용자도 이해 가능한 설명형 화면”에 가깝게 설계한다.
